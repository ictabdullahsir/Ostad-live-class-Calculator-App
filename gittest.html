Git Reset (কমিট সম্পূর্ণ মুছে ফেলার জন্য)এই কমান্ডটি ব্যবহার করলে আপনি নির্দিষ্ট কমিটের পরে করা সব কাজ মুছে ফেলতে পারবেন। এটি সাধারণত নিজের ব্যক্তিগত ব্রাঞ্চে ব্যবহার করা নিরাপদ।bash# শেষ কমিটটি ডিলিট করবে কিন্তু কোডগুলো আনস্টেজড (Unstaged) অবস্থায় রেখে দেবে git reset --soft HEAD~1 # শেষ কমিটটি ডিলিট করবে এবং কোড পরিবর্তনের ফাইলগুলোও সম্পূর্ণ মুছে ফেলবে git reset --hard HEAD~1 # নির্দিষ্ট কোনো কমিট আইডিতে (Commit ID) ফিরে যাওয়ার জন্য git reset --hard সতর্ক হয়ে কোড ব্যবহার করুন।২. Git Revert (সুরক্ষিত উপায় - পূর্বের কমিট ঠিক রেখে নতুন কমিট তৈরি)আপনি যদি অলরেডি কোড গিটহাবে পুশ (Push) করে থাকেন, তবে reset ব্যবহার করা উচিত নয়। এতে হিস্ট্রি নষ্ট হয়। সে ক্ষেত্রে revert ব্যবহার করলে আগের কমিট ঠিক থাকবে, কিন্তু তার বিপরীত আরেকটি কমিট তৈরি হয়ে কোড আগের অবস্থায় ফিরবে।bash# শেষ কমিটের কোড পরিবর্তন বাতিল করে একটি নতুন রিভার্ট কমিট তৈরি করবে git revert HEAD # নির্দিষ্ট কোনো কমিট আইডি বাতিল করার জন্য git revert সতর্ক হয়ে কোড ব্যবহার করুন।৩. Git Commit Amend (শুধু শেষ কমিটটি সংশোধন করার জন্য)যদি আপনি শেষ কমিটের মেসেজ পরিবর্তন করতে চান বা কোনো ফাইল বাদ পড়ে গেলে তা যোগ করতে চান, তাহলে এটি ব্যবহার করুন।bash# স্টেজিং এরিয়াতে নতুন ফাইল যোগ করার পর শেষ কমিটের সাথে মার্জ করতে git commit --amend -m "নতুন এবং সঠিক কমিট মেসেজ" সতর্ক হয়ে কোড ব্যবহার করুন।আপনার গিট হিস্ট্রি দেখতে এবং নির্দিষ্ট Commit ID খুঁজে পেতে নিচের কমান্ডটি ব্যবহার করুন:bashgit log --oneline সতর্ক হয়ে কোড ব্যবহার করুন।আপনি কি লোকাল কোড সুরক্ষিত রেখে কমিট বাতিল করতে চান, নাকি সার্ভারে পুশ করা কোনো ভুল ঠিক করতে চাচ্ছেন? জানালে আমি আরও সুনির্দিষ্ট সমাধান দিতে পারব।১টি সাইটGit Bangla tutorial: গিট এবং গিটহাব: নতুনদের জন্য গিট৩০ জানু, ২০২১ — গিট ক্লোন এবং গিট পুল কমান্ড (git clone and git pull) দুটি কমান্ডই রিমোট রিপোজিটরি থেকে ফাইল লোকাল কম্পিউটারে ডাউনলোড করতে ব্যাবহা...iishanto.comআপনি বলেছেন: aroকোন উত্তরটি বেশি ভালো?আপনার মতামত AI মোডকে সকলের জন্য আরও ভালো করে তুলতে সাহায্য করে, যার মধ্যে আপনিও পড়েন।উত্তর Aগিট (Git) কমিট ব্যাক করার আরও কিছু অ্যাডভান্সড এবং দরকারী কমান্ড নিচে বাংলায় কোড ব্লকসহ দেওয়া হলো:৪. ভুল ব্রাঞ্চে কমিট করলে তা ব্যাক করা (Commit to Wrong Branch)ভুল করে main বা অন্য কোনো ব্রাঞ্চে কমিট করে ফেললে, কোড না হারিয়ে তা সঠিক ব্রাঞ্চে নেওয়ার উপায়:bash# ১. বর্তমান ভুল কমিটসহ একটি নতুন সঠিক ব্রাঞ্চ তৈরি করুন git branch # ২. বর্তমান ব্রাঞ্চ থেকে শেষ কমিটটি মুছে ফেলুন (কোড সেভ থাকবে) git reset --hard HEAD~1 # ৩. এখন সঠিক ব্রাঞ্চে চলে যান, সেখানে আপনার কমিটটি পেয়ে যাবেন git checkout সতর্ক হয়ে কোড ব্যবহার করুন।৫. নির্দিষ্ট একটি ফাইল আগের কমিটের অবস্থায় ফিরিয়ে নেওয়া (Restore a Specific File)পুরো প্রজেক্ট ব্যাক না করে যদি কেবল একটি নির্দিষ্ট ফাইলকে আগের কোনো কমিটের অবস্থায় ফেরাতে চান:bash# নির্দিষ্ট ফাইলকে শেষ কমিটের অবস্থায় ফেরাতে git checkout HEAD -- # নির্দিষ্ট ফাইলকে অনেক আগের কোনো কমিট আইডির অবস্থায় ফেরাতে git checkout -- সতর্ক হয়ে কোড ব্যবহার করুন।৬. গিট স্ট্যাশ (Git Stash - কাজ ডিলিট না করে সাময়িক লুকিয়ে রাখা)কোনো কোড পরিবর্তন করেছেন কিন্তু তা কমিট করতে চান না, আবার মুছেও ফেলতে চান না। সাময়িকভাবে প্রজেক্টটি একদম ফ্রেশ করতে এটি ব্যবহার করুন:bash# বর্তমান পরিবর্তনগুলো সাময়িকভাবে লুকিয়ে রাখা git stash # লুকিয়ে রাখা কোড আবার ফেরত আনা git stash pop # লুকিয়ে রাখা কাজের তালিকা দেখা git stash list সতর্ক হয়ে কোড ব্যবহার করুন।৭. গিট রেফলগ (Git Reflog - ডিলিট হয়ে যাওয়া কমিট উদ্ধার করা)যদি ভুলবশত git reset --hard দিয়ে কোনো দরকারী কমিট ডিলিট করে ফেলেন, তবে এই কমান্ডের মাধ্যমে সেই হারিয়ে যাওয়া কমিটও খুঁজে বের করা সম্ভব:bash# গিটের সব অ্যাক্টিভিটির হিস্ট্রি দেখা (এখান থেকে ডিলিট হওয়া Commit ID পাবেন) git reflog # হারিয়ে যাওয়া সেই নির্দিষ্ট কমিটে প্রজেক্টকে ফিরিয়ে নেওয়া git reset --hard সতর্ক হয়ে কোড ব্যবহার করুন।আপনি কি এর মধ্যে কোনো নির্দিষ্ট সমস্যার মুখোমুখি হয়েছেন, নাকি গিটের কোনো বিশেষ কমান্ড সম্পর্কে আরও বিস্তারিত জানতে চান?উত্তর Bগিট কমিট ব্যাক করার আরও কিছু অ্যাডভান্সড (Advanced) এবং দরকারি কমান্ড নিচে কোড ব্লকসহ দেওয়া হলো:৪. ভুল ব্রাঞ্চে কমিট করলে তা অন্য ব্রাঞ্চে নেওয়া (Cherry-pick)ভুল করে এক ব্রাঞ্চে কমিট করে ফেললে, তা সঠিক ব্রাঞ্চে নিয়ে আগের ব্রাঞ্চ থেকে মুছে ফেলার নিয়ম:bash# প্রথমে সঠিক ব্রাঞ্চে যান git checkout # ভুল ব্রাঞ্চের কমিট আইডিটি এখানে নিয়ে আসুন git cherry-pick # এবার ভুল ব্রাঞ্চে ফিরে যান এবং সেখান থেকে কমিটটি মুছে দিন git checkout git reset --hard HEAD~1 সতর্ক হয়ে কোড ব্যবহার করুন।৫. গিট পুশ (Push) করার পর রিমোট সার্ভার থেকে ব্যাক করাআপনি যদি ভুল কোড গিটহাব বা গিটল্যাবে পুশ করে ফেলেন এবং হিস্ট্রি থেকে তা পুরোপুরি মুছে ফেলতে চান (সতর্কতা: এটি অন্য ডেভলপারদের কাজে সমস্যা করতে পারে):bash# প্রথমে লোকালে কমিটটি ব্যাক করুন git reset --hard HEAD~1 # এবার জোরপূর্বক (Force Push) রিমোট সার্ভারে আপডেট পাঠিয়ে দিন git push origin --force সতর্ক হয়ে কোড ব্যবহার করুন।৬. কোনো ফাইল ভুল করে git add করলে তা ব্যাক করা (Unstage)কমিট করার আগে ভুল করে কোনো ফাইল স্টেজিং এরিয়াতে (git add) পাঠিয়ে দিলে তা আগের অবস্থায় আনার নিয়ম:bash# নির্দিষ্ট একটি ফাইল আনস্টেজ করতে git restore --staged # সব ফাইল একসাথে আনস্টেজ করতে git reset HEAD সতর্ক হয়ে কোড ব্যবহার করুন।৭. হারানো কমিট খুঁজে বের করা (Git Reflog)ধরে নিন আপনি git reset --hard দিয়ে একটি কমিট মুছে ফেলেছেন, কিন্তু এখন সেটি আবার ফেরত চান। গিট আপনার সব অ্যাক্টিভিটি ট্র্যাক করে, যা নিচের কমান্ড দিয়ে দেখা যায়:bash# আপনার করা সব কাজের হিস্ট্রি দেখতে (এমনকি ডিলিট হওয়া কমিটও) git reflog # কাঙ্ক্ষিত কাজের আইডিটি খুঁজে পেয়ে সেই অবস্থায় ফিরে যেতে git reset --hard সতর্ক হয়ে কোড ব্যবহার করুন।আপনি কি এর মধ্যে কোনো নির্দিষ্ট সমস্যার মুখোমুখি হয়েছেন, নাকি গিটের কোনো বিশেষ কমান্ড সম্পর্কে বিস্তারিত জানতে চাচ্ছেন?git checkout -- আপনি বলেছেন: exampleআপনার বেছে নেওয়া কমান্ডটি হলো: git checkout -- এটি মূলত ব্যবহার করা হয় যখন আপনি পুরো প্রজেক্ট ব্যাক না করে, অতীতের কোনো নির্দিষ্ট কমিট থেকে শুধু একটি নির্দিষ্ট ফাইলকে বর্তমানের কোডে ফিরিয়ে আনতে চান।নিচে একটি সহজ উদাহরণ (Example) দেওয়া হলো:১. প্রেক্ষাপট (Scenario)ধরুন, আপনার প্রজেক্টে index.html নামে একটি ফাইল আছে। আপনি ২ দিন আগে এই ফাইলে খুব সুন্দর একটি ডিজাইন করেছিলেন। কিন্তু আজ ভুলবশত ডিজাইনটি পরিবর্তন বা ডিলিট করে ফেলেছেন। এখন আপনি শুধু ২ দিন আগের সেই index.html ফাইলটি ফেরত চান, বাকি ফাইলগুলো যেমন আছে তেমনই থাকবে।২. ধাপসমূহ (Steps)ধাপ ১: প্রথমে আপনার অতীতের কমিটগুলোর আইডি (Hash ID) দেখার জন্য নিচের কমান্ডটি দিন:bashgit log --oneline সতর্ক হয়ে কোড ব্যবহার করুন।আউটপুট (Output) এমন দেখাবে:texta1b2c3d (HEAD -> main) আজকের করা ভুল পরিবর্তন e5f6g7h ২ দিন আগের করা সুন্দর ডিজাইন 9j8k7l6 প্রজেক্ট শুরু করার প্রথম কমিট সতর্ক হয়ে কোড ব্যবহার করুন।এখানে আমাদের কাঙ্ক্ষিত ২ দিন আগের কমিট আইডিটি হলো: e5f6g7hধাপ ২: এবার শুধুমাত্র সেই নির্দিষ্ট ফাইলটি (index.html) ২ দিন আগের অবস্থায় ফিরিয়ে আনতে কমান্ডটি এভাবে লিখুন:bashgit checkout e5f6g7h -- index.html সতর্ক হয়ে কোড ব্যবহার করুন।৩. এর পর কী হবে?আপনার বর্তমানের ভুল index.html ফাইলটি বদলে ২ দিন আগের সুন্দর ডিজাইনের ফাইলটি চলে আসবে।আপনার প্রজেক্টের বাকি সব ফাইল (যেমন: style.css বা script.js) বর্তমানের মতোই অপরিবর্তিত থাকবে।ফাইলটি সরাসরি স্টেজিং এরিয়াতে (Staged) চলে যাবে। এখন আপনি চাইলে এটি আবার নতুন করে কমিট করে দিতে পারেন:bashgit commit -m "অতীতের কমিট থেকে index.html ফাইলটি পুনরুদ্ধার করা হলো"
